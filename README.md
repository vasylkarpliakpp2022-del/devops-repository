# Екзаменаційна робота: Технології DevOps (Частина 2)

Цей репозиторій містить рішення екзаменаційного завдання з побудови комплексного CI/CD конвеєра для автоматизованого розгортання інфраструктури, конфігурації сервера та деплою застосунку за допомогою сучасних DevOps-практик.

## 🗂 Структура репозиторію

- `/task1/` — **Terraform**: Інфраструктура як код (IaC). Скрипти для розгортання ресурсів у DigitalOcean (VPC, Firewall, Droplet Ubuntu 24, S3 Bucket для зберігання `tfstate`).
- `/task2/` — **Ansible**: Конфігурація сервера. Playbooks для автоматичного встановлення Kubernetes, Minikube, Helm та налаштування проксі.
- `/task3/` — **Kubernetes & App Deploy**: Маніфести для розгортання застосунку в Minikube у різних середовищах (`dev` та `release`).
- `/.github/workflows/` — **GitHub Actions**: Набір CI/CD конвеєрів для повної автоматизації процесів.

## 🚀 Реалізовані CI/CD Конвеєри

Усі процеси автоматизовані за допомогою **GitHub Actions** і розділені на логічні етапи. Використовуються практики безпеки: усі конфіденційні дані (токени, ключі SSH) приховані через GitHub Secrets.

1. **Task 1: Terraform Deploy** - Виконує розгортання інфраструктури. 
   - Містить обов'язковий крок `Terraform Plan` (dry-run) перед застосуванням змін.
   - Стан інфраструктури (`tfstate`) безпечно зберігається у хмарному бакеті (S3).

2. **Task 2: Ansible Deploy**
   - Налаштовує створений сервер.
   - Містить обов'язковий крок перевірки `ansible-playbook --check` (dry-run).

3. **Task 3: App Deploy to K8s**
   - Збирає Docker-образи для середовищ `dev` (з гілки `develop`) та `release` (з тегу `1.0.0`).
   - Відправляє образи до Docker Hub.
   - Перевіряє маніфести (`--dry-run=client`) та виконує деплой у відповідні неймспейси.
   - **Особливість:** Автоматично створює фоновий **Systemd-сервіс** для стабільного прокидання портів (8002 для `dev`, 8003 для `release`).

## 🌟 Бонусні завдання (Додаткові бали)

В рамках проекту реалізовано додаткові конвеєри (запускаються вручну через `workflow_dispatch` для безпеки):

- 🧨 **Task 1: Terraform DESTROY (Danger Zone)** — Окремий пайплайн для швидкого та безпечного видалення всієї створеної інфраструктури, щоб уникнути зайвих витрат у хмарі.
- ⏪ **Task 3: App Rollback** — Пайплайн для миттєвого відкату застосунку (Zero-Downtime Rollback) на попередню стабільну версію в Kubernetes. Підтримує вибір середовища (`dev` або `release`).

- **Провайдер:** DigitalOcean

- **Configuration Management:** Ansible
- **Containerization & Orchestration:** Docker, Kubernetes (Minikube), Helm
